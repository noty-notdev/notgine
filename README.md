

# **notgine**

> _"**not** an en**gine**, not in itself_".
>
> - **noty !f(x)**.


The **notgine** is not a videogame engine, not in itself.
Although it may sound contradictory, I see it in the long term
as _"an engine of engines"_. A fundamental tool that help
build medium-sized and/or large engines,
depending on the developer's needs.

Although, I do want to create something useful enough that
a certain number of people will use it as a graphics engine
--in a way --, the main purpose of it is make it modular enough
so that the developer can override its behavior in code,
without modifying the source files.
